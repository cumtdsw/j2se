现在感觉aviator还是太笨重，现在越来越追求简单的东西。aviator足以淘汰的因素：
1. 自定义方法需要额外接口去实现，无法直接在脚本中定义。
2. 设计上过多潜规则，例如exec方法后面可以带参数，但是java参数是无法获知其名称的，
   这意味着你需要在expression中包含变量名，又潜在需要把exec参数保持顺序，这就是他妈坑爹。
   例如：
   		double a = 3, b = 4, c = 5;
		Object ret = AviatorEvaluator.exec("a*b/c", a, b, c);
		System.out.println(ret);
  输出的值是2.4，对的。
  过了一段时间，发现公式写错了或者需要修改，于是乎把公式改成，a*c/b，结果还是2.4，够坑爹吧。
  更过分的时，如果公式很长，然后一个参数还用多次，顺序还比较混乱，那就跪了吧，垃圾的设计！
3. 没有if else、do while等语句，没有赋值语句，不够灵活。
