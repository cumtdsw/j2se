这个框架是简单的hibernate类型框架，自动创建表、自动映射插入、修改、删除

这样框架适合于简单的项目，因为太过于自动化，使得优化的自由度变低、难度变大。

关于设计:
1. Bean类的表和字段注解设计非常不错，轻量级，免配置。
2. DAO的泛型设计是jfinal参考的那种，总体上也不错。
如果有通用的dao设计，我觉得会更好，这个通用的dao根据对象的class自动获取注解来更新。
从泛型设计上看，一个表的id只能有一个字段，这个可能是限制。
3. 没有insertNotNull UpdateNotNull这样的接口
4. 和spring jdbcTemplate的整合效果未知。
